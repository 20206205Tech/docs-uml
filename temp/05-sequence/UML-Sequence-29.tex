\paragraph{Sơ đồ tuần tự Pha khởi tạo giao dịch mua gói thuê bao VIP}

Pha này mô tả bước User chọn gói thuê bao VIP trên Web Frontend, hệ thống tạo Transaction ở trạng thái \texttt{PENDING}, tạo Subscription mới nếu người dùng chưa có subscription, đăng ký job timeout trên Redis, sau đó gọi Payment Gateway để sinh đường link thanh toán.

\subparagraph{Mô tả chi tiết}
\begin{itemize}
\item \textbf{Gửi yêu cầu mua gói thuê bao VIP:} User chọn nâng cấp VIP, Frontend gửi \texttt{POST /subscriptions/purchase} qua Kong kèm Bearer Token.
\item \textbf{Chuẩn bị dữ liệu thanh toán:} Payment Service kiểm tra plan, subscription hiện tại, tạo subscription mới ở trạng thái \texttt{PENDING} nếu cần và tạo transaction mới.
\item \textbf{Tạo link thanh toán:} Service đăng ký job timeout trên Redis, gọi Payment Gateway để lấy \texttt{payment\_url}, rồi trả link về Frontend để chuyển hướng User.
\item \textbf{Xử lý timeout:} Nếu sau 10 phút transaction vẫn ở trạng thái \texttt{PENDING}, Payment Timeout Processor cập nhật Transaction và Subscription liên quan thành \texttt{EXPIRED}; nếu transaction đã được xử lý thì bỏ qua job timeout.
\end{itemize}
