\paragraph{Sơ đồ tuần tự Luồng Xác nhận Email kích hoạt tài khoản}

Luồng nghiệp vụ này mô tả quá trình người dùng nhấp vào liên kết xác thực từ hòm thư điện tử cá nhân để kích hoạt tài khoản, chuyển hướng qua API Gateway (Kong) và nhận token JWT để đăng nhập chính thức.

\subparagraph{Các thành phần tham gia}

\begin{table}[H]
\centering
\renewcommand{\arraystretch}{1.3}
\begin{tabular}{|l|l|p{5cm}|}
\hline
\textbf{Thành phần} & \textbf{Phân loại} & \textbf{Vai trò trong hệ thống} \\ \hline
Guest & Actor & Khách vãng lai thực hiện nhấp chọn liên kết xác thực. \\ \hline
Web Frontend & Component & Trang Callback xử lý trích xuất token từ URL Hash và chuyển hướng trang. \\ \hline
API Gateway (Kong) & Component & Cổng trung gian tiếp nhận liên kết xác thực từ người dùng và chuyển tiếp tới Auth Service. \\ \hline
Auth Service & Microservice & Dịch vụ xác thực cập nhật trạng thái hoạt động của tài khoản và cấp cặp token JWT qua luồng redirect. \\ \hline
Database & Database & Cập nhật thời gian kích hoạt tài khoản (\texttt{email\_confirmed\_at}). \\ \hline
\end{tabular}
\caption{Các thành phần tham gia luồng Xác nhận Email kích hoạt tài khoản}
\end{table}

\subparagraph{Mô tả tổng quan}

\begin{itemize}
\item Guest mở hòm thư cá nhân và nhấp chọn liên kết xác thực được gửi từ hệ thống (liên kết được trỏ qua API Gateway Kong).
\item API Gateway chuyển tiếp yêu cầu xác thực tài khoản tới Auth Service.
\item Auth Service kiểm tra tính hợp lệ của token xác thực và cập nhật cơ sở dữ liệu để kích hoạt tài khoản (\texttt{email\_confirmed\_at = NOW}).
\item Sau khi cập nhật thành công, Auth Service trả về phản hồi HTTP 302 Redirect, chuyển hướng trình duyệt của người dùng quay trở lại trang callback của Web Frontend (\texttt{/auth/callback}) kèm theo cặp token JWT trong phần URL Hash.
\item Web Frontend trích xuất Access Token và Refresh Token từ URL Hash, lưu trữ cục bộ và chuyển hướng người dùng đăng nhập thành công vào giao diện chính (\texttt{/chat}).
\end{itemize}
