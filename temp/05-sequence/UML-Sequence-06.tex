\paragraph{Sơ đồ tuần tự Luồng Đăng nhập bằng Email \& Password}

Luồng nghiệp vụ này mô tả quá trình người dùng sử dụng tài khoản đã được kích hoạt trước đó để đăng nhập trực tiếp vào hệ thống bằng phương thức POST trả về dữ liệu định dạng JSON.

\subparagraph{Các thành phần tham gia}

\begin{table}[H]
\centering
\renewcommand{\arraystretch}{1.3}
\begin{tabular}{|l|l|p{5cm}|}
\hline
\textbf{Thành phần} & \textbf{Phân loại} & \textbf{Vai trò trong hệ thống} \\ \hline
Guest & Actor & Khách vãng lai đăng nhập bằng tài khoản sẵn có. \\ \hline
Web Frontend & Component & Giao diện tiếp nhận thông tin và lưu trữ token đăng nhập. \\ \hline

Auth Service (Supabase) & Microservice & Xác thực thông tin tài khoản và trả trực tiếp dữ liệu token dạng JSON. \\ \hline
Supabase Database & Database & Kiểm tra thông tin người dùng và trạng thái kích hoạt tài khoản. \\ \hline
\end{tabular}

\end{table}

\subparagraph{Mô tả tổng quan}

\begin{itemize}
\item Guest điền địa chỉ Email và mật khẩu, chọn nút Đăng nhập trên Web Frontend.
\item Web Frontend thực hiện cuộc gọi POST trực tiếp qua API Gateway (Kong) đến endpoint lấy token của Auth Service.
\item Auth Service truy vấn Database để kiểm tra thông tin đăng nhập và đảm bảo tài khoản đã được xác nhận email (\texttt{email\_confirmed\_at} khác null).
\item Nếu thông tin hợp lệ, Auth Service sinh cặp mã token JWT (Access Token và Refresh Token) gửi trực tiếp về API Gateway dưới dạng phản hồi JSON.
\item API Gateway chuyển tiếp phản hồi JSON về Web Frontend. Frontend thực hiện lưu trữ mã token cục bộ (Cookie/LocalStorage) và chuyển hướng người dùng vào trang giao diện chính (\texttt{/chat}).
\end{itemize}
