\paragraph{Sơ đồ tuần tự Chia sẻ và Thu hồi cuộc trò chuyện (Share \& Revoke Flow)}

Luồng nghiệp vụ này mô tả quy trình người dùng tạo liên kết chia sẻ cuộc trò chuyện pháp lý của mình ra ngoài công cộng, quy trình khách truy cập (Guest/Viewer) xem nội dung cuộc trò chuyện đó dưới dạng chỉ đọc (Read-only), và quy trình người dùng thu hồi liên kết chia sẻ.

\subparagraph{Mô tả chi tiết}
\begin{itemize}
\item \textbf{Tạo liên kết chia sẻ}: User chọn chia sẻ, Frontend gửi `POST /shares` qua Kong Gateway tới Conversation Service. Dịch vụ tạo token ngẫu nhiên, lưu vào Neon Database ở trạng thái `active` và trả về URL dạng `/shares/{token}`.
\item \textbf{Viewer truy cập}: Viewer mở liên kết, Frontend gọi API `GET /shares/{token}`. Conversation Service kiểm tra nếu token còn hiệu lực (`active`), sẽ truy vấn lịch sử hội thoại tương ứng trong Database và trả về dạng chỉ đọc (không cho phép gửi tin nhắn mới).
\item \textbf{Thu hồi liên kết}: Khi không muốn chia sẻ nữa, User bấm thu hồi. Frontend gửi yêu cầu `DELETE /shares/{token}` qua Gateway đến Conversation Service để đánh dấu trạng thái token thành `revoked` (hoặc xóa khỏi Database), vô hiệu hóa hoàn toàn liên kết chia sẻ.
\end{itemize}
